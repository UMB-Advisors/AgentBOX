# Quick Task 260528-96q: MBOX-162 S1 Multi-Account Concurrency Benchmark

**One-liner:** N-account classify+draft concurrency harness with injectable deps, discriminated-union S1 verdict (classify p95 < 5000ms AND peak ≤ 4.0 GiB from addendum §6 / DR-45), and deterministic Vitest suite covering scheduling, p95, boundary gating, and memory peak.

## What Was Built

### Task 1 — `dashboard/lib/eval/concurrency-bench.ts` (commit 357c059)

Core benchmark lib. Key design choices:

- `runConcurrencyBench(accounts, traces, mode, deps?)` — runs N-account classify+draft in `serialized` (safe T2) or `concurrent` (T3 candidate) mode. Classify uses an injectable `classifyFn` (default = real Ollama `/api/generate`); draft calls `runBakeOffOnTrace` with an injectable `fetchFn`.
- Memory sampler polls an injectable `readUsedMemGiB()` via injectable `setIntervalFn` / `clearIntervalFn`. Takes one immediate t0 sample before the interval starts, one final sample after all accounts settle. Peak = `Math.max(...samples, 0)`.
- `evaluateS1Verdict({ classifyP95Ms, peakMemGiB })` — discriminated union: `{ verdict: 'pass' }` iff `classifyP95Ms < 5000 && peakMemGiB <= 4.0`; else `{ verdict: 'fail', breaches: S1Breach[] }`. Boundary: exactly 5000ms → FAIL (strict `<`); exactly 4.0 GiB → PASS (`<=`). Null classify p95 → breach (cannot prove < 5000).
- `percentile` (Type-7, re-exported) — identical semantics to `bake-off.ts`'s private copy.
- `defaultReadUsedMemGiB()` — reads `/proc/meminfo`, computes `(MemTotal - MemAvailable) / KB_PER_GIB`; returns 0 on any failure (never throws).
- All types use `strict` mode, no `any`, discriminated unions, `unknown` + guards where needed.

### Task 2 — `dashboard/scripts/concurrency-bench.ts` (commit 44fe451)

CLI runner mirroring `bake-off-harness.ts` style:

- `#!/usr/bin/env -S npx tsx` shebang, `isDirect` execution guard, hand-rolled `parseArgs` with `parseIntStrict` / `parseFloatStrict`, `--help` / `-h` exit 0.
- `--accounts` (default 3 / `MBOX_BENCH_ACCOUNTS`), `--classify-base-url` (/ `OLLAMA_BASE_URL`), `--mode both|serialized|concurrent` (default `both`), `--out eval/results` (default).
- Inline trace-set loader (~40 LOC; mirrors `bake-off-harness.ts` shape).
- Writes `{run-tag}.summary.json` (provenance + `ConcurrencyBenchResult[]`) and `{run-tag}.verdict.md` (markdown table + DR-45 interpretation line) under gitignored `eval/results/`.
- No deps overrides in CLI — real classify/draft fetch, real `/proc/meminfo` sampler, real clock/timers. Injection is for tests only.

### Task 3 — `dashboard/test/lib/concurrency-bench.test.ts` (commit f28545f)

17 deterministic tests, zero hardware/network/real-clock dependence:

1. **`evaluateS1Verdict` boundary table** (7 cases): all five spec'd cases plus two extras covering the extremes. Including: 4999/4.0 → pass; 5000/4.0 → fail(`classify_p95`); 4999/4.0001 → fail(`peak_memory`); 5000/4.0001 → fail(both); null/4.0 → fail(`classify_p95`).
2. **`percentile`** (5 cases): empty→null, single→value, [100..500] p50=300/p95=480, p0, p100.
3. **Scheduling** (2 cases): serialized asserts acct-0 classify+done recorded before acct-1 classify fires; concurrent asserts both classify calls fire before any draft resolves (controllable draft stall via Promise).
4. **p95 computation** (1 case): scripted per-account latency maps; asserts per-account p95 and overall flattened p95 via Type-7 interpolation.
5. **Memory peak** (2 cases): `vi.useFakeTimers()` + `advanceTimersByTimeAsync`; injectable `readUsedMemGiB` returning scripted sequence `[1.0, 3.5, 2.0, ...]`; asserts `peak_mem_gib === 3.5` and `peak_mem_gib === Math.max(...mem_samples_gib)`.

## Follow-up: typecheck fix (commit 82004dd)

`npm run typecheck` (`tsc --noEmit`, which includes `**/*.ts` so test files are typechecked) was exiting 1 with 12 errors in `test/lib/concurrency-bench.test.ts` — vitest had passed only because esbuild strips types. Root cause: `ConcurrencyBenchDeps.setIntervalFn`/`clearIntervalFn` were typed as the global `typeof setInterval`/`clearInterval`, whose primary overload returns `number`; the Node-`Timeout`-returning test mocks failed assignment (TS2322), which cascaded into implicit-`any` mock params (TS7006) and `never`-call-site errors (TS2349).

Fix (in the lib so the dep surface is environment-agnostic): added `export type IntervalHandle = ReturnType<typeof setInterval>` and retyped the deps as `setIntervalFn?: (handler: () => void, ms: number) => IntervalHandle` and `clearIntervalFn?: (handle: IntervalHandle) => void` (real `setInterval`/`clearInterval` satisfy this). Test mocks given explicit param types + `IntervalHandle` returns; the concurrent-mode draft resolvers moved into a ref object so CFA doesn't narrow the post-await read to `never`. No `any`, no new casts beyond the pre-existing `as unknown as IntervalHandle` no-op handle stubs.

Verification: `npm run typecheck` exits 0; `npx vitest run test/lib/concurrency-bench.test.ts` 17 passed; full `npx vitest run` 723 passed / 165 skipped, no regressions.
