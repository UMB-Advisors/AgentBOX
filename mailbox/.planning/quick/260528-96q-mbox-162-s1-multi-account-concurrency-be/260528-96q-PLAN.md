Build the MBOX-162 spike **S1** harness: a multi-account T2 concurrency benchmark
that measures whether a Jetson can classify+draft for N (default 3) accounts —
in both `serialized` and `concurrent` scheduling modes — without breaching the
S1 gate from `docs/addendum-mailbox-multi-account-v0_1-2026-05-20.md` §6 and the
DR-45 kill criterion: **classify p95 < 5000ms AND peak memory ≤ 4.0 GiB**.

here — the real measurement run executes later on the live M1 appliance in an
operator-scheduled quiet window. Every external dependency (classify fetch, draft
fetch via `runBakeOffOnTrace`, memory sampler, clock, interval timer, timeouts) is
INJECTABLE so the unit tests are fully deterministic.

Purpose: produce evidence to promote (or kill) DR-45 — whether T2 multi-account is
serialized-only or can run concurrent. SM-90 ("Classify p95 < 5s sustained with N
accounts active") is the metric this harness measures.

Output: `concurrency-bench.ts` lib + CLI + test, mirroring the style, header-comment
density, injectable-deps shape, and discriminated-union conventions of the existing
