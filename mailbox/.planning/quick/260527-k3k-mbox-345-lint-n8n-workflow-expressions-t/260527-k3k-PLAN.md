<objective>
Build a regression-guard lint over `n8n/workflows/*.json` that catches the MBOX-344 bug
class: a node inserted on a main-input path silently blanks a downstream node's `$json.*`
inputs because `$json` now resolves to the inserted node's (different) output.

The MBOX-344 incident: `MailBOX-Send`'s `Gmail Reply` read `{{ $json.message_id }}` /
`{{ $json.draft_body }}`, but the `Acquire Send Lock` Postgres node (RETURNING `id` only)
was inserted between `Load Draft` and `Gmail Reply` (via the `Lock Acquired?` IF). `$json`
resolved to `{ id }` → both fields went empty → Gmail 400 "Invalid id value." Fix was
repointing to `{{ $('Load Draft').item.json.* }}`.

Purpose: prevent silent re-introduction of this class with zero new CI infra.
Output: a pure analyzer in `dashboard/lib/`, a vitest guard that runs inside the already-green
`dashboard (typecheck + test)` gate, and a documented `$('Node').item.json.*` convention.

Scope guard (per design guidance): SIMPLEST reliable catch — NOT a general-purpose n8n
data-lineage analyzer. Keep false positives at zero by only flagging when there's positive
evidence the field is missing (immediate predecessor is a Postgres `executeQuery` whose
`RETURNING`/`SELECT` field set is determinable and does NOT include the consumed field).
When predecessor output can't be determined, do NOT flag — the floor assertion covers the
critical path regardless.
</objective>

A workflow JSON has:
```
{
  "name": string,                  // e.g. "MailBOX-Send"
  "nodes": Array<{
    name: string,                  // e.g. "Gmail Reply", "Load Draft", "Acquire Send Lock"
    type: string,                  // e.g. "n8n-nodes-base.postgres", "n8n-nodes-base.gmail"
    parameters: Record<string, unknown>  // nested object; expression strings live here
  }>,
  "connections": Record<string,    // keyed by SOURCE node name
    { main: Array<Array<{ node: string, type: "main", index: number }>> }
  >
}
```

Predecessor map (what feeds a node's MAIN input): invert `connections`. For each source
node S, for each entry T in S.connections.main[*][*] with type==="main", T.node's
main-input predecessor is S. (A node may have one main predecessor in these linear-ish
workflows; if multiple, keep the set.)

Expression strings to scan (recursively over `parameters` values that are strings):
- `={{ $json.message_id }}`            → reads field `message_id`
- `={{ $json["draft_body"] }}`         → reads field `draft_body`
- `={{ $json.body.draft_id }}`         → reads field path `body.draft_id` (top-level field = `body`)
- `={{ $('Load Draft').item.json.x }}` → cross-node ref, NOT a bare $json read → IGNORE

Postgres node output fields (for `type === "n8n-nodes-base.postgres"` with
`parameters.operation === "executeQuery"`): parse `parameters.query` (SQL string).
- `RETURNING id;`               → output field set { id }
- `RETURNING id, status`        → { id, status }
- `SELECT a, b AS alias FROM …` → { a, alias }  (last alias / column name wins)
Real example — `Acquire Send Lock` query ends in `RETURNING id;` → output = { id }.
`Load Draft` is `SELECT d.id AS draft_id, … m.message_id, …` → output includes
{ draft_id, draft_subject, draft_body, status, sent_gmail_message_id, inbox_message_id,
message_id, thread_id, from_addr, to_addr, original_subject }.

Floor target node (MailBOX-Send):
```
{ name: "Gmail Reply", type: "n8n-nodes-base.gmail",
  parameters: { messageId: "={{ $('Load Draft').item.json.message_id }}",
                message:   "={{ $('Load Draft').item.json.draft_body }}", ... } }
```
Both MUST reference `$('Load Draft')` (or any `$('…')` cross-node ref) — never bare `$json`.

Create `dashboard/lib/n8n-expr-lint.ts` — a PURE, dependency-free TS module (no fs, no I/O;
takes parsed workflow objects so it's trivially unit-testable). Follow `dashboard/lib`
conventions: `interface` over `type` for object shapes, no `any` (use `unknown` + narrowing),
named exports. Export:

  - `interface N8nNode { name: string; type: string; parameters: Record<string, unknown>; }`
  - `interface N8nWorkflow { name: string; nodes: N8nNode[]; connections: Record<string, { main?: Array<Array<{ node: string; type: string; index: number }>> }>; }`
  - `interface Violation { workflow: string; node: string; field: string; predecessor: string; suggestion: string; }`
  - `buildMainInputPredecessors(wf: N8nWorkflow): Map<string, Set<string>>`
      Invert `connections`: for each source S and each target T in S.main[*][*] where
      type==="main", add S to predecessors.get(T.node).
  - `extractJsonFieldReads(value: unknown): Set<string>`
      Recurse through objects/arrays. For each string, regex-scan for bare $json reads:
      `/\$json(?:\.([A-Za-z_$][\w$]*)|\[(['"])([^'"]+)\2\])/g`. Take the FIRST path segment
      as the top-level field (so `$json.body.draft_id` → `body`). CRITICAL: do NOT match
      `$('…').item.json.…` — anchor on `$json` NOT preceded by `)` / `.` of a node ref. The
      simplest robust approach: first strip/skip any `$('…').item.json...` substrings (or only
      match `$json` where the 6 chars are a standalone token: preceded by start, whitespace,
      `{`, `(`, `=` — never by `.`). Add a unit test proving `$('X').item.json.y` is ignored.
  - `postgresReturningFields(query: string): Set<string> | null`
      (a) If `/\bRETURNING\b/i` present: take the substring after RETURNING up to `;`/end,
      split on top-level commas, for each item take the alias after ` AS ` else the last
      `.`-segment identifier. Return the Set.
      (b) Else if a single top-level `SELECT … FROM` is parseable: split the select list on
      top-level commas, derive each output name (alias after AS, else last dotted segment),
      return the Set. Bail to `null` if the column list contains `*` or is otherwise ambiguous.
      (c) Else return `null` (undeterminable — caller must NOT flag).
      Keep the SQL parsing intentionally shallow; this is a heuristic for the RETURNING-id
      signal, not a SQL parser. Comments in the query (lines starting `--`) must be stripped
      before scanning so the `Acquire Send Lock` comment block doesn't confuse the regex.
  - `lintWorkflow(wf: N8nWorkflow): Violation[]`
      For each node N: preds = buildMainInputPredecessors(wf).get(N.name). If preds has
      exactly one entry P and P is a Postgres node (`type === 'n8n-nodes-base.postgres'`,
      `parameters.operation === 'executeQuery'`) with `postgresReturningFields(P.parameters.query)`
      non-null → outSet. Then for each field F in `extractJsonFieldReads(N.parameters)`:
      if !outSet.has(F) → push Violation{ workflow: wf.name, node: N.name, field: F,
      predecessor: P.name, suggestion: `use $('<earlier-node>').item.json.${F}` }.
      Otherwise (multiple preds, non-Postgres pred, or null outSet) → skip (no flag).
