# Phase 260512-080 Plan 01: Toast UNDO + Visibility-Gated Polling Summary

Backend route + audit hook for undoing a fresh reject, plus a toast confirmation with a 5s UNDO action and visibility-aware 30s polling on the queue.

## What shipped

**C1 — `ea85b2c` — `feat(drafts): undo-reject route + state_transitions audit hook (STAQPRO-331 #9)`**

- New `POST /api/drafts/[id]/undo-reject` route that, in one transaction:
  - Sets session GUCs `mailbox.actor='operator'` and `mailbox.transition_reason='undo_reject'` so the migration-009 trigger captures the audit row.
  - Flips `mailbox.drafts.status` from `rejected` → `pending` (guarded; 409 on any other source state).
  - Deletes the LATEST `mailbox.draft_feedback` row for the draft via a Kysely `eb('id', 'in', selectFrom(...))` subquery — preserves any earlier rejection-cycle feedback.
- `undoRejectBodySchema = z.object({}).strict()` added to `lib/schemas/drafts.ts`. Confirmed `parseJson` normalizes missing/invalid bodies to `{}` so the route accepts empty/missing body cleanly.
- `getLatestTransition(draftId)` helper added to `test/helpers/db.ts` reading `from_status / to_status / actor / reason` ordered by `transitioned_at DESC` (column names per migration 009 — `transitioned_at`, not `changed_at`).
- 4 new vitest cases under `describe('POST /api/drafts/[id]/undo-reject', ...)`:
  1. Happy path: reject → 200 + feedback row exists → undo → 200 + status `pending` + feedback removed.
  2. 409 guard: undo on `pending` returns 409, draft unchanged, no feedback rows created.
  3. Audit row: reject → undo → `getLatestTransition` shows `from='rejected'`, `to='pending'`, `actor='operator'`, `reason='undo_reject'`. Proves the migration-009 trigger picked up the session GUCs.
  4. Idempotency: reject → undo (200) → undo (409). No additional feedback rows touched.

**C2 — `d6c95d5` — `feat(queue): toast confirmation with UNDO + visibility-gated polling (STAQPRO-331 #9, #11)`**

- `Toast` extended with optional `action: { label, onClick }` and `durationMs` props; Esc dismisses globally via a `window.addEventListener('keydown')` in the same effect. `aria-live="polite"` added. Existing two call sites (approve/edit success, error toasts) unaffected.
- `ToastMsg` in QueueClient widened to carry `action` + `durationMs`; existing `<Toast {...toast} />` spread carries them through.
- `fireReject` re-implemented inline (no longer a wrapper over `fireAction`) so on success it sets a toast with `text: "Rejected · <REJECT_REASON_LABELS[reason_code]>"`, `durationMs: 5000`, and an `action.onClick: () => fireUndoReject(draft.id)`.
- New `fireUndoReject(draftId)` helper POSTs `'{}'` to `/api/drafts/:id/undo-reject`, drops the local `removed` mark so the draft reappears in `visibleActive`, surfaces `Reject undone` toast on success or a structured error toast on failure (409 window expired etc).
- Polling effect rewritten to gate on `document.visibilityState === 'visible'` for setInterval ticks AND add a `visibilitychange` listener that fires `fetchData(false)` on return-to-foreground.
- `REJECT_REASON_LABELS: Record<RejectReasonCode, string>` added to `lib/types.ts` as the single SoT for any UI label surface (toast + future analytics/report tooling).
- Approve toast left alone (no UNDO — Gmail Reply already fired at n8n).

## Deviations from plan

- **Kysely subquery syntax** — used the explicit `eb` callback form per the planner's NOTE (`.where(({ eb, selectFrom }) => eb('id', 'in', selectFrom(...)))`). The inline `.where('id', 'in', subquery)` form did not appear necessary to attempt; the eb form typechecks cleanly. No deviation in behavior.
- **`parseJson` empty-body handling** — confirmed `parseJson` already normalizes missing/invalid JSON to `{}` (lib/middleware/validate.ts line 42). `z.object({}).strict()` accepts `{}` and rejects unknown keys. The planner's `.optional()` fallback was not needed.
- **AbortController per fetch** — DEFERRED per scope guidance. Documented in code comment + decisions block.
- **is-new 3s pulse on new drafts** — DEFERRED per scope (planner explicitly excluded it).
- **biome auto-fixes** — biome flagged import sort order on `lib/types` import in QueueClient (`type DraftWithMessage` before `REJECT_REASON_LABELS`) and a brace-placement preference in the new test cases. Applied via `npx biome check --write components/QueueClient.tsx test/routes/drafts.test.ts`. Pure formatting; included in C2.

## Out-of-scope deferrals

- **is-new pulse animation** — 3s green flash on freshly-polled rows; UX nice-to-have, not required for #11.
- **AbortController + fetchData(signal)** — would prevent overlapping in-flight polls but requires threading a signal through `fetchData` and both inner fetches; current last-write-wins is idempotent enough.
- **Re-running on-appliance smoke** — out of scope for this executor; PR reviewer / deploy operator runs it post-merge.
- **Cleanup of RejectPopover.tsx to consume REJECT_REASON_LABELS** — left for a future small PR; the popover's REJECT_REASONS list is richer (label + hint) than the bare label map, and consolidating would touch unrelated UI surface.
