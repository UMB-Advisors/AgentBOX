# Brain Graph Tab — PRD (v0.1.0)

**Status:** In build · **Date:** 2026-06-01 · **Owner:** Dustin

**Resumes:** the paused "gbrain knowledge-graph viz via Understand-Anything" plan.

## TL;DR

Add a **"Brain Graph"** primary nav item to the AgentBOX dashboard, directly
below **Scheduled Actions**, that renders an interactive Understand-Anything (UA)
graph of the **gbrain** knowledge base. The graph is a periodic static snapshot:
an adapter dumps gbrain `pages`+`links` → a UA `knowledge-graph.json`; a prebuilt
static UA "demo-mode" bundle renders it; the dashboard embeds that bundle in a
same-origin iframe (mirroring the existing Incoming-Messages embed). gbrain lives
on **mailbox2** (the deploy box), so the adapter + refresh run there; this
workstation (`bob-TB250-BTC`) only builds source + a sample-data proof.

## Why this shape

| Concern | Decision | Rationale |
|---|---|---|
| Nav placement | New primary sidebar item `/graph`, after Scheduled Actions | User-chosen (sibling, not a sub-tab) |
| gbrain → UA mapping | Custom adapter (pages→nodes, links→edges) | gbrain is not a Karpathy wiki; off-the-shelf `understand-knowledge` won't ingest it |
| Embed mechanism | Static UA **demo-mode** bundle, served same-origin under `/graph-app/`, iframed | UA demo mode (`VITE_DEMO_MODE`) drops the token gate and reads a configurable graph URL → no sidecar server, no token plumbing. Same pattern as the inbox `/dashboard/*` embed |
| Freshness | Periodic snapshot regenerated by a scheduled action on mailbox2 | gbrain enrichment (links) is async; live render is unnecessary and fits the "Scheduled Actions" home |
| Where it runs | Adapter + bundle refresh on mailbox2 (gbrain host); source built here | gbrain (`~/gbrain-src`, `~/.gbrain`, bun) exists only on mailbox2 |

## Architecture

```
mailbox2 (gbrain host)                      AgentBOX dashboard (hermes web_server)
──────────────────────                      ─────────────────────────────────────
~/.gbrain (pages + links)                   web/src/pages/GraphPage.tsx
   │  engine.listPages()/getLinks()            └─ iframe src="/graph-app/"
   ▼  (tools/gbrain-graph-export.ts, bun)    web/src/App.tsx
knowledge-graph.json  ───────────────────►     ├─ route  /graph → GraphPage
   │                                            └─ nav    "Brain Graph" after Scheduled Actions
   ▼ dropped into graph_app/                 hermes_cli/web_server.py
UA static demo bundle (graph_app/)             └─ mount /graph-app → StaticFiles(graph_app, html=True)
   (built once by tools/build-graph-app.sh)        serves index.html + assets + knowledge-graph.json
```

UA graph contract (validated by `@understand-anything/core` `validateGraph`):
- top-level **`project`** object (required), `nodes[]`, `edges[]`.
- node: `{ id, type:"document", name, filePath:<slug> }` (one per gbrain page).
- edge: `{ source, target, type, direction:"forward" }` — gbrain `link_type`
  mapped to a UA `EdgeType` (`mentions`→`related`, `source`→`cites`, else `related`).

## Deploy runbook (mailbox2)

1. **Gate — verify gbrain links populated:** `bun run ~/gbrain-src/src/cli.ts graph-query <slug> --depth 2` returns edges (paused-plan gate; enrichment is async via the Minions queue). If empty, the graph renders nodes-only until enrichment runs.
2. Build the UA demo bundle (once, or when UA upgrades): `tools/build-graph-app.sh` → produces `graph_app/`. Ship `graph_app/` next to `web_server.py` in `~/.hermes/hermes-agent/hermes_cli/` (or set `HERMES_GRAPH_APP_DIST`).
3. Generate the graph: `bun run ~/gbrain-src/src/tools/gbrain-graph-export.ts --out ~/.hermes/hermes-agent/hermes_cli/graph_app/knowledge-graph.json` (ABSOLUTE bun path `~/.bun/bin/bun` — gbrain CLI is not on a non-login PATH).
4. Deploy the rebuilt SPA bundle + `web_server.py` to mailbox2; `systemctl --user restart hermes-dashboard`.
5. Schedule step 3 (e.g. daily) as a Scheduled Action so the snapshot tracks gbrain growth.
